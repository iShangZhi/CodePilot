# Plugin: git-diff

通用 git diff 执行与结构化输出能力。

## 能力描述

对指定工程目录执行 git diff，将原始 diff 转换为结构化变更报告，供下游 Agent 消费。

## 执行步骤

```bash
# 1. 获取已暂存 + 未暂存的全量变更
git -C {project_path} diff HEAD

# 2. 获取新增未追踪文件列表
git -C {project_path} status --short
```

## 输出格式

```markdown
### 变更文件清单
| 文件路径 | 变更类型 | 新增行 | 删除行 |
|---------|---------|--------|--------|
| ...     | 新增/修改/删除 | +N | -N |

### 变更摘要
- 变更文件总数：N
- 新增文件：N
- 修改文件：N
- 删除文件：N

### 关键变更片段
（每个文件取前 30 行 diff，超出截断）
```

## 注意事项

- 仅读取 diff，不做任何提交或暂存操作
- 输出行数上限：100 行，超出截断并注明「已截断，完整 diff 请查看原始文件」
- 二进制文件变更只记录文件名，不展示内容

## 使用方 Agent 调用约定

```
调用 git-diff，目标路径：{project_path}
输出结构化变更报告，写入上下文
```
